# 第 08 章：新增、修改、刪除資料 

> **學習目標**
> - 使用 INSERT 新增資料
> - 使用 UPDATE 修改資料
> - 使用 DELETE 刪除資料
> - 了解交易（Transaction）的重要性

---

## 8.1 INSERT — 新增資料

### 基本語法

```sql
INSERT INTO 表格名稱 (欄位1, 欄位2, ...)
VALUES (值1, 值2, ...);
```

### 範例：新增一位顧客

```sql
INSERT INTO customers (name, email, city, joined_date)
VALUES ('李大華', 'lihua@example.com', '台南', '2026-04-29');
```

驗證：
```sql
SELECT * FROM customers WHERE name = '李大華';
```

### 插入時欄位順序

欄位名稱和 VALUES 的值**必須一一對應**：

```sql
--  可以只插入部分欄位（其他使用預設值或 NULL）
INSERT INTO customers (name, email)
VALUES ('測試用戶', 'test@example.com');
-- city 和 joined_date 會使用預設值
```

> ️ 有 `NOT NULL` 限制的欄位，必須提供值，否則會報錯。

---

## 8.2 一次插入多筆資料

```sql
INSERT INTO categories (category_name) VALUES
    ('旅遊'),
    ('烹飪'),
    ('科技');
```

這比分三次執行更有效率。

---

## 8.3 INSERT OR REPLACE / INSERT OR IGNORE

SQLite 提供衝突處理選項：

```sql
-- 若 email 已存在（違反 UNIQUE），忽略這次插入
INSERT OR IGNORE INTO customers (name, email, city)
VALUES ('重複用戶', 'ming@example.com', '台北');

-- 若衝突，刪除舊資料並插入新資料
INSERT OR REPLACE INTO customers (name, email, city)
VALUES ('陳小明', 'ming@example.com', '高雄');
```

## 8.4 UPDATE — 修改資料

### 基本語法

```sql
UPDATE 表格名稱
SET 欄位1 = 新值1, 欄位2 = 新值2, ...
WHERE 條件;
```

> ️ **非常重要**：`UPDATE` 不加 `WHERE` 會修改**所有列**！  
> 執行前，先用 `SELECT` 確認要修改的資料。

### 範例：修改一本書的價格

```sql
-- ① 先確認
SELECT title, price FROM products WHERE title = '活著';
-- 活著 | 280

-- ② 再修改
UPDATE products
SET price = 295
WHERE title = '活著';

-- ③ 驗證
SELECT title, price FROM products WHERE title = '活著';
-- 活著 | 295
```

### 修改多個欄位

```sql
UPDATE products
SET
    price = 310,
    stock = stock + 10   -- 庫存增加 10
WHERE product_id = 3;
```

---

## 8.5 UPDATE 常見錯誤

```sql
--  危險！沒有 WHERE，全部書的價格都會變成 999
UPDATE products SET price = 999;

--  安全寫法：加上 WHERE 限制範圍
UPDATE products SET price = 999 WHERE product_id = 1;
```

### 用子查詢作為條件

```sql
-- 將所有科幻類書籍的庫存補充到 50
UPDATE products
SET stock = 50
WHERE category_id = (
    SELECT category_id FROM categories
    WHERE category_name = '科幻'
)
AND stock < 50;
```

## 8.6 DELETE — 刪除資料

### 基本語法

```sql
DELETE FROM 表格名稱
WHERE 條件;
```

> ️ **非常危險**：`DELETE` 不加 `WHERE` 會刪除**所有資料**！  
> 執行前，用 `SELECT` 確認要刪除的範圍。

### 範例：刪除已取消的訂單

```sql
-- ① 先確認要刪除的資料
SELECT * FROM orders WHERE status = 'cancelled';

-- ② 刪除
DELETE FROM orders WHERE status = 'cancelled';
```

### 範例：刪除特定顧客

```sql
DELETE FROM customers WHERE customer_id = 8;
```

---

## 8.7 DELETE vs TRUNCATE

| 指令 | 說明 | 可回復 | SQLite 支援 |
|------|------|--------|-------------|
| `DELETE FROM 表格 WHERE ...` | 刪除特定列 |  用交易 |  |
| `DELETE FROM 表格` | 刪除所有列（保留表格）|  用交易 |  |
| `TRUNCATE TABLE 表格` | 快速清空（不可回復）|  |  SQLite 不支援 |

SQLite 中清空表格的方式：

```sql
DELETE FROM 表格名稱;               -- 刪除所有資料
-- 若想重置 AUTOINCREMENT 計數器：
DELETE FROM sqlite_sequence WHERE name = '表格名稱';
```

## 8.8 交易（Transaction）

**交易**確保多條 SQL 語句要嘛全部成功，要嘛全部回復：

```sql
BEGIN;                  -- 開始交易

UPDATE products SET stock = stock - 1 WHERE product_id = 1;
INSERT INTO order_items (order_id, product_id, quantity, unit_price)
    VALUES (11, 1, 1, 320);

COMMIT;                 -- 全部成功，確認提交
-- 或
ROLLBACK;               -- 發生問題，回復到 BEGIN 之前
```

### 為什麼需要交易？

假設「下訂單時扣除庫存」的操作：

```
步驟 1：扣除庫存  ← 成功了
步驟 2：建立訂單  ← 失敗！→ 庫存少了但訂單沒建立，資料不一致！
```

用交易包住兩個步驟，若步驟 2 失敗，步驟 1 也會回復。

---

## 重點整理

| 指令 | 用途 | 必要子句 |
|------|------|----------|
| `INSERT INTO 表格 (欄位) VALUES (值)` | 新增資料 | — |
| `UPDATE 表格 SET 欄位=值 WHERE 條件` | 修改資料 | ️ WHERE |
| `DELETE FROM 表格 WHERE 條件` | 刪除資料 | ️ WHERE |
| `BEGIN; ... COMMIT;` | 交易（批次操作）| — |
| `BEGIN; ... ROLLBACK;` | 交易（回復操作）| — |

### 操作資料的安全習慣

1. **修改/刪除前，先用 SELECT 確認**
2. **一定要加 WHERE 子句**（除非真的要全部處理）
3. **重要操作用交易包住**
4. **在測試環境先驗證，再到正式環境執行**
