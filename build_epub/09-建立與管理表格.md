## 9.1 CREATE TABLE — 建立表格

### 基本語法

```sql
CREATE TABLE 表格名稱 (
    欄位名稱  資料類型  [限制條件],
    欄位名稱  資料類型  [限制條件],
    ...
);
```

### 範例：建立一張新的「評論」表格

```sql
CREATE TABLE reviews (
    review_id   INTEGER  PRIMARY KEY AUTOINCREMENT,
    product_id  INTEGER  NOT NULL,
    customer_id INTEGER  NOT NULL,
    rating      INTEGER  NOT NULL CHECK (rating BETWEEN 1 AND 5),
    comment     TEXT,
    created_at  TEXT     NOT NULL DEFAULT (DATETIME('now'))
);
```

## 9.2 SQLite 資料類型

SQLite 使用**動態型別**系統，主要有五種親和性：

| 類型 | 關鍵字 | 說明 | 範例 |
|------|--------|------|------|
| **整數** | `INTEGER` | 整數值 | `1`, `42`, `-5` |
| **浮點數** | `REAL` | 小數、浮點數 | `3.14`, `99.9` |
| **文字** | `TEXT` | 字串（含日期） | `'台北'`, `'2026-01-01'` |
| **二進位** | `BLOB` | 原始二進位資料 | 圖片、檔案 |
| **空值** | `NULL` | 無值 | — |

### 常見型別別名

SQLite 接受以下常見寫法（自動對應到上述類型）：

| 寫法 | 對應類型 | 說明 |
|------|----------|------|
| `INT`, `INTEGER` | INTEGER | 整數 |
| `REAL`, `FLOAT`, `DOUBLE` | REAL | 浮點數 |
| `TEXT`, `VARCHAR(n)`, `CHAR(n)` | TEXT | 字串（SQLite 不限長度）|
| `NUMERIC`, `DECIMAL(p,s)` | 依值決定 | 數值 |
| `BOOLEAN` | INTEGER | SQLite 用 0/1 表示 |
| `DATE`, `DATETIME` | TEXT | 日期（SQLite 用字串儲存）|

## 9.3 限制條件（Constraints）

建立表格時，可以加入規則限制資料：

| 限制 | 說明 |
|------|------|
| `PRIMARY KEY` | 主鍵，唯一且不能為 NULL |
| `AUTOINCREMENT` | 自動遞增整數（搭配 INTEGER PRIMARY KEY）|
| `NOT NULL` | 不允許空值 |
| `UNIQUE` | 值必須唯一 |
| `DEFAULT 值` | 沒有給值時使用預設值 |
| `CHECK (條件)` | 值必須符合條件 |
| `REFERENCES 表格(欄位)` | 外鍵，關聯到其他表格 |

### 範例：建立帶限制的表格

```sql
CREATE TABLE products_v2 (
    product_id   INTEGER PRIMARY KEY AUTOINCREMENT,
    title        TEXT    NOT NULL,                    -- 不能為空
    isbn         TEXT    UNIQUE,                      -- 書號唯一
    price        REAL    NOT NULL CHECK (price > 0),  -- 價格必須 > 0
    stock        INTEGER NOT NULL DEFAULT 0,          -- 預設庫存 0
    author_id    INTEGER REFERENCES authors(author_id),  -- 外鍵
    is_active    INTEGER NOT NULL DEFAULT 1           -- 1=上架, 0=下架
);
```

## 9.4 IF NOT EXISTS

避免重複建立表格報錯：

```sql
CREATE TABLE IF NOT EXISTS reviews (
    review_id INTEGER PRIMARY KEY AUTOINCREMENT,
    ...
);
```

## 9.6 ALTER TABLE — 修改表格

SQLite 的 `ALTER TABLE` 功能有限，只支援：

### 新增欄位

```sql
ALTER TABLE reviews
ADD COLUMN updated_at TEXT;
```

### 重新命名表格

```sql
ALTER TABLE reviews
RENAME TO book_reviews;
```

### 重新命名欄位（SQLite 3.25.0+）

```sql
ALTER TABLE book_reviews
RENAME COLUMN comment TO review_text;
```

> ️ SQLite **不支援** `DROP COLUMN`（3.35.0 以上版本才支援有限的刪除欄位）。  
> 若需要刪除欄位，通常採用「建新表→複製資料→改名」的方式。

---

## 9.7 DROP TABLE — 刪除表格

```sql
DROP TABLE reviews;

-- 若存在才刪除（避免報錯）
DROP TABLE IF EXISTS reviews;
```

> ️ 刪除表格**無法復原**，請謹慎操作。

## 9.8 索引（Index）

**索引**類似書本的目錄，讓資料庫快速找到資料，不必掃描整張表。

### 何時需要索引？

- 常用於 `WHERE` 條件的欄位
- 常用於 `JOIN` 連接的欄位
- 常用於 `ORDER BY` 的欄位

### 建立索引

```sql
-- 單欄位索引
CREATE INDEX idx_products_price
ON products (price);

-- 多欄位組合索引
CREATE INDEX idx_products_cat_price
ON products (category_id, price);

-- 唯一索引（同時確保資料唯一）
CREATE UNIQUE INDEX idx_customers_email
ON customers (email);
```

### 查看索引

```sql
-- 查看某表格的所有索引
PRAGMA index_list(products);
```

### 刪除索引

```sql
DROP INDEX IF EXISTS idx_products_price;
```

### 索引的代價

- 索引會佔用磁碟空間
- 每次 `INSERT`、`UPDATE`、`DELETE` 都要同步更新索引
- **不要對每個欄位都建索引**，只對常用查詢的欄位建立

## 9.9 設計良好表格的原則

### 正規化（Normalization）基本概念

**第一正規化（1NF）**：每個欄位只存一個值

```sql
--  壞設計：interests 存多個值
customers: interests = '閱讀,登山,烹飪'

--  好設計：用關聯表格
customer_interests: customer_id | interest
                    1           | 閱讀
                    1           | 登山
```

**避免資料重複**：將重複的資料移到獨立表格，透過外鍵關聯

```sql
--  壞設計：每張訂單都重複存顧客城市
orders: customer_name='陳小明', customer_city='台北', ...

--  好設計：顧客資料存在 customers，透過 customer_id 關聯
orders: customer_id=1, ...
```

## 重點整理

| 語法 | 用途 |
|------|------|
| `CREATE TABLE 表格 (欄位 類型 限制)` | 建立表格 |
| `CREATE TABLE IF NOT EXISTS ...` | 避免重複建立報錯 |
| `ALTER TABLE 表格 ADD COLUMN 欄位 類型` | 新增欄位 |
| `ALTER TABLE 表格 RENAME TO 新名稱` | 重新命名表格 |
| `DROP TABLE 表格` | 刪除表格 |
| `DROP TABLE IF EXISTS 表格` | 安全刪除 |
| `CREATE INDEX 索引名 ON 表格(欄位)` | 建立索引 |
| `DROP INDEX 索引名` | 刪除索引 |

### 常用限制條件

| 限制 | 效果 |
|------|------|
| `PRIMARY KEY AUTOINCREMENT` | 自動遞增主鍵 |
| `NOT NULL` | 必填欄位 |
| `UNIQUE` | 唯一值 |
| `DEFAULT 值` | 預設值 |
| `CHECK (條件)` | 值的規則 |
| `REFERENCES 表格(欄位)` | 外鍵 |
