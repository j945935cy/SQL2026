# 第 05 章：聚合函數與分組 {#ch05}

![聚合與分組圖](../images/ch05.png)

> **學習目標**
> - 使用 COUNT、SUM、AVG、MAX、MIN 計算統計數據
> - 使用 GROUP BY 分組統計
> - 使用 HAVING 篩選分組後的結果
> - 理解 WHERE 與 HAVING 的差異

---

## 5.1 什麼是聚合函數？

**聚合函數**（Aggregate Function）對多列資料進行計算，回傳**單一結果**。

例如：「書店共有幾本書？」、「所有書的平均價格是多少？」

```
輸入：多列資料  →  聚合函數  →  輸出：單一數值
```

---

## 5.2 五大聚合函數

| 函數 | 說明 | 範例 |
|------|------|------|
| `COUNT(*)` | 計算列數（含 NULL）| 共有幾筆資料 |
| `COUNT(欄位)` | 計算非 NULL 的筆數 | 有填寫的筆數 |
| `SUM(欄位)` | 加總 | 所有訂單金額加總 |
| `AVG(欄位)` | 平均值 | 書籍平均價格 |
| `MAX(欄位)` | 最大值 | 最貴的書 |
| `MIN(欄位)` | 最小值 | 最便宜的書 |

---

## 5.3 COUNT — 計算筆數

### 計算所有書籍數量

```sql
SELECT COUNT(*) AS 書籍總數
FROM products;
```
```
書籍總數
────────
10
```

### COUNT(*) vs COUNT(欄位)

```sql
-- COUNT(*) 計算所有列（含 NULL）
SELECT COUNT(*) FROM products;          -- 10

-- COUNT(欄位) 只計算非 NULL 的值
SELECT COUNT(published_year) FROM products;  -- 若有 NULL 則小於 10
```

### 條件計算：庫存書籍數量

```sql
SELECT COUNT(*) AS 有庫存書籍數
FROM products
WHERE stock > 0;
```

---

## 5.4 SUM — 加總

### 計算所有書籍的總庫存

```sql
SELECT SUM(stock) AS 總庫存
FROM products;
```

### 計算某分類的總庫存

```sql
SELECT SUM(stock) AS 文學小說總庫存
FROM products
WHERE category_id = 1;
```

---

## 5.5 AVG — 平均值

### 計算書籍平均售價

```sql
SELECT ROUND(AVG(price), 2) AS 平均售價
FROM products;
```

> `ROUND(數值, 小數位數)` 四捨五入到指定小數位

```
平均售價
────────
335.0
```

---

## 5.6 MAX 和 MIN — 最大 / 最小值

```sql
SELECT
    MAX(price) AS 最高售價,
    MIN(price) AS 最低售價,
    MAX(price) - MIN(price) AS 價差
FROM products;
```
```
最高售價  最低售價  價差
────────  ────────  ────
430       260       170
```

### 同時使用多個聚合函數

```sql
SELECT
    COUNT(*)           AS 書籍總數,
    SUM(stock)         AS 總庫存,
    ROUND(AVG(price), 0) AS 平均售價,
    MAX(price)         AS 最高售價,
    MIN(price)         AS 最低售價
FROM products;
```

---

## 5.7 GROUP BY — 分組統計

`GROUP BY` 將資料依照某個欄位分組，再對每組進行聚合計算：

```sql
SELECT 分組欄位, 聚合函數(...)
FROM 表格
GROUP BY 分組欄位;
```

### 範例：各城市的顧客數量

```sql
SELECT city, COUNT(*) AS 顧客數
FROM customers
GROUP BY city;
```
```
city  顧客數
────  ──────
台中  1
台南  1
台北  3
桃園  1
新竹  1
高雄  1
```

### 範例：各分類的書籍數量和平均售價

```sql
SELECT
    category_id,
    COUNT(*)              AS 書籍數,
    ROUND(AVG(price), 0)  AS 平均售價,
    SUM(stock)            AS 總庫存
FROM products
GROUP BY category_id;
```
```
category_id  書籍數  平均售價  總庫存
───────────  ──────  ────────  ──────
1            6       315       180
2            2       425       72
3            1       350       40
5            1       360       75
```

### GROUP BY 的規則

> ⚠️ **重要**：`SELECT` 中的欄位，要嘛在 `GROUP BY` 中出現，要嘛必須放在聚合函數中。

```sql
-- ❌ 錯誤：title 沒有在 GROUP BY 中
SELECT category_id, title, COUNT(*)
FROM products
GROUP BY category_id;

-- ✅ 正確
SELECT category_id, COUNT(*), MAX(title)
FROM products
GROUP BY category_id;
```

---

## 5.8 GROUP BY 多個欄位

依多個欄位組合分組：

```sql
SELECT
    nationality AS 國籍,
    -- （假設 products 有作者資訊）
    COUNT(*) AS 作者數
FROM authors
GROUP BY nationality;
```

---

## 5.9 HAVING — 篩選分組結果

`WHERE` 篩選的是**原始列**，`HAVING` 篩選的是**分組後的結果**：

```sql
SELECT 分組欄位, 聚合函數(...)
FROM 表格
GROUP BY 分組欄位
HAVING 聚合條件;
```

### 範例：找出顧客數超過 1 人的城市

```sql
SELECT city, COUNT(*) AS 顧客數
FROM customers
GROUP BY city
HAVING COUNT(*) > 1;
```
```
city  顧客數
────  ──────
台北  3
```

### 範例：平均售價超過 350 元的分類

```sql
SELECT
    category_id,
    ROUND(AVG(price), 0) AS 平均售價
FROM products
GROUP BY category_id
HAVING AVG(price) > 350;
```

---

## 5.10 WHERE vs HAVING

| 比較點 | WHERE | HAVING |
|--------|-------|--------|
| 篩選對象 | 原始列（GROUP BY 之前）| 分組後的結果 |
| 能否用聚合函數 | ❌ 不能 | ✅ 可以 |
| 執行順序 | 較早 | 較晚 |

```sql
-- ✅ 兩者可以同時使用
SELECT
    category_id,
    COUNT(*)   AS 書籍數,
    AVG(price) AS 平均售價
FROM products
WHERE stock > 0          -- ← WHERE：先篩掉庫存為 0 的書
GROUP BY category_id
HAVING COUNT(*) >= 2;    -- ← HAVING：只看書籍數 >= 2 的分類
```

---

## 5.11 完整查詢執行順序

```
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
```

```sql
SELECT
    category_id,
    COUNT(*)             AS 書籍數,   -- 5. 計算要顯示的欄位
    ROUND(AVG(price),0)  AS 平均售價
FROM products                         -- 1. 來源表格
WHERE stock > 0                       -- 2. 篩選原始列
GROUP BY category_id                  -- 3. 分組
HAVING COUNT(*) >= 2                  -- 4. 篩選分組結果
ORDER BY 平均售價 DESC                -- 6. 排序
LIMIT 3;                              -- 7. 限制筆數
```

---

## 重點整理

| 語法 | 用途 |
|------|------|
| `COUNT(*)` | 計算列數 |
| `COUNT(欄位)` | 計算非 NULL 筆數 |
| `SUM(欄位)` | 加總 |
| `AVG(欄位)` | 平均 |
| `MAX(欄位)` | 最大值 |
| `MIN(欄位)` | 最小值 |
| `GROUP BY 欄位` | 依欄位分組 |
| `HAVING 聚合條件` | 篩選分組結果 |
| `ROUND(數值, N)` | 四捨五入到 N 位小數 |
