## 1.1 為什麼需要資料庫？

想像你在管理一間書店，有幾百種書、幾千位顧客、每天幾十筆訂單。  
如果用 Excel 管理，會遇到：

- 資料重複（同一位顧客出現在多張訂單工作表）
- 更新麻煩（改顧客電話要找遍所有工作表）
- 難以查詢（「台北的顧客今年共買了哪些書？」）

**資料庫**（Database）解決這些問題：
- 資料集中儲存，不重複
- 可以快速查詢、篩選、排序
- 多人同時存取也不衝突

---

## 1.2 什麼是關聯式資料庫？

**關聯式資料庫**（Relational Database）是目前最常用的資料庫類型。  
它把資料存在**表格**（Table）裡，就像 Excel 的工作表。

### 核心概念

| 名詞 | 說明 | Excel 對應 |
|------|------|------------|
| **資料庫**（Database） | 資料的容器 | 整個 Excel 檔案 |
| **表格**（Table） | 同類資料的集合 | 一張工作表 |
| **欄位**（Column / Field） | 資料的屬性 | 一個欄（A、B、C…） |
| **列**（Row / Record） | 一筆資料 | 一行資料 |
| **主鍵**（Primary Key） | 唯一識別每列的欄位 | 流水號欄 |
| **外鍵**（Foreign Key） | 指向另一表格主鍵的欄位 | VLOOKUP 的連結 |

### 範例：customers 表格

```
customer_id │ name   │ email               │ city
────────────┼────────┼─────────────────────┼──────
     1      │ 陳小明  │ ming@example.com    │ 台北
     2      │ 林雅婷  │ yating@example.com  │ 台中
     3      │ 王志偉  │ wei@example.com     │ 高雄
```

- `customer_id` 是**主鍵**：每位顧客有唯一編號
- `name`、`email`、`city` 是各種**欄位**
- 每一行是一筆**記錄**

---

## 1.3 什麼是 SQL？

**SQL**（Structured Query Language，結構化查詢語言）是與關聯式資料庫溝通的語言。

你用 SQL 告訴資料庫：
- 「給我所有台北的顧客」→ **查詢**
- 「新增一位顧客」→ **新增**
- 「把這本書的價格改成 350 元」→ **修改**
- 「刪除這張訂單」→ **刪除**

SQL 語句分四大類：

| 類型 | 英文全名 | 代表指令 | 用途 |
|------|----------|----------|------|
| **DQL** | Data Query Language | `SELECT` | 查詢資料 |
| **DML** | Data Manipulation Language | `INSERT` / `UPDATE` / `DELETE` | 操作資料 |
| **DDL** | Data Definition Language | `CREATE` / `ALTER` / `DROP` | 定義結構 |
| **DCL** | Data Control Language | `GRANT` / `REVOKE` | 權限管理 |

> 初學者 90% 的時間在用 DQL（查詢），其次是 DML（操作）。

---

## 1.4 什麼是 SQLite？

SQLite 是一個**輕量級**、**免安裝伺服器**的關聯式資料庫，特點：

- ✅ 整個資料庫就是一個 `.db` 檔案
- ✅ 不需要安裝資料庫伺服器
- ✅ 完全免費、開源
- ✅ 廣泛用於手機 App、桌面軟體、小型網站
- ✅ SQL 語法與主流資料庫高度相容

### SQLite vs 其他資料庫

| 特性 | SQLite | MySQL | PostgreSQL |
|------|--------|-------|------------|
| 需要伺服器 | ❌ 不需要 | ✅ 需要 | ✅ 需要 |
| 適合初學練習 | ✅ 非常適合 | ✅ 適合 | ✅ 適合 |
| 適合生產環境 | 小型 App | 中大型網站 | 企業級 |

## 1.5 安裝 SQLite

### Windows

1. 前往 [https://www.sqlite.org/download.html](https://www.sqlite.org/download.html)
2. 下載 **Precompiled Binaries for Windows** 中的 `sqlite-tools-win-x64-*.zip`
3. 解壓縮，將資料夾路徑加入系統 PATH 環境變數
4. 開啟命令提示字元，輸入 `sqlite3 --version` 確認安裝成功

### macOS（使用 Homebrew）

```bash
brew install sqlite
sqlite3 --version
```

### Linux（Ubuntu / Debian）

```bash
sudo apt update
sudo apt install sqlite3
sqlite3 --version
```

### 推薦圖形介面工具

| 工具 | 平台 | 說明 |
|------|------|------|
| [DB Browser for SQLite](https://sqlitebrowser.org/) | Win / Mac / Linux | 免費、適合初學者 |
| [TablePlus](https://tableplus.com/) | Win / Mac | 付費，介面優美 |
| VS Code + SQLite Viewer 外掛 | 所有平台 | 輕量，直接在編輯器中查看 |

## 1.6 建立第一個資料庫

開啟終端機（命令提示字元），進入本書目錄，執行：

```bash
# 建立資料庫並載入範例資料
sqlite3 bookstore.db < sample-db/setup.sql
```

接著進入 SQLite 互動介面：

```bash
sqlite3 bookstore.db
```

你會看到：

```
SQLite version 3.x.x
Enter ".help" for usage hints.
sqlite>
```

試試第一條 SQL：

```sql
SELECT name, city FROM customers;
```

輸出結果：

```
陳小明|台北
林雅婷|台中
王志偉|高雄
吳美玲|台北
黃建宏|新竹
張宇航|台南
劉思穎|台北
周靜怡|桃園
```

🎉 恭喜！你已經成功執行了第一條 SQL 查詢！

### 常用 SQLite 命令

| 命令 | 說明 |
|------|------|
| `.tables` | 列出所有表格 |
| `.schema customers` | 顯示 customers 表格結構 |
| `.headers on` | 顯示欄位名稱標題 |
| `.mode column` | 以欄位對齊方式顯示 |
| `.quit` | 離開 SQLite |

建議執行：
```
sqlite> .headers on
sqlite> .mode column
```
讓輸出更易讀。

## 1.7 本書範例資料庫總覽

本書使用「書店管理系統」，共 6 張表格：

```
customers（顧客）
  └── orders（訂單）
        └── order_items（訂單明細）
                └── products（書籍）
                      ├── categories（分類）
                      └── authors（作者）
```

執行 `.tables` 確認所有表格已建立：

```sql
.tables
-- authors  categories  customers  order_items  orders  products
```

---

## 重點整理

- 資料庫將資料有組織地儲存在表格中，解決 Excel 管理的缺點
- 關聯式資料庫由表格、欄位、列組成，透過主鍵和外鍵建立關聯
- SQL 是與資料庫溝通的標準語言，分 DQL、DML、DDL、DCL
- SQLite 是輕量免伺服器的資料庫，非常適合學習使用
- 本書所有範例使用「書店管理系統」資料庫
