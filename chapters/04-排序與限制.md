# 第 04 章：排序與限制

> **學習目標**
> - 使用 ORDER BY 對查詢結果排序
> - 多欄位排序與排序方向
> - 使用 LIMIT 限制回傳筆數
> - 使用 OFFSET 實現分頁查詢

---

## 4.1 ORDER BY — 排序

預設查詢結果的順序是資料庫儲存的順序，不一定有意義。  
使用 `ORDER BY` 指定**排序欄位**：

```sql
SELECT 欄位 FROM 表格
WHERE 條件
ORDER BY 排序欄位;
```

### 範例：書籍依價格由低到高排列

```sql
SELECT title, price
FROM products
ORDER BY price;
```

結果：
```
title         price
──────────    ─────
活著          280
死水微瀾      260   ← 最低
傳說          290
挪威的森林    320
...
```

---

## 4.2 ASC 和 DESC

| 關鍵字 | 說明 | 預設 |
|--------|------|------|
| `ASC` | 升冪（小→大）| ✅ 預設 |
| `DESC` | 降冪（大→小）| 需指定 |

### 範例：書籍依價格由高到低

```sql
SELECT title, price
FROM products
ORDER BY price DESC;
```

結果：
```
title     price
────────  ─────
黑暗森林  430
三體      420
海邊的卡夫卡 380
...
```

### 字串排序

字串按字典序（Unicode）排列：

```sql
SELECT name FROM customers
ORDER BY name ASC;
-- 依中文筆劃/Unicode 排序
```

---

## 4.3 多欄位排序

當第一個欄位有相同值時，用第二個欄位決定順序：

```sql
SELECT 欄位 FROM 表格
ORDER BY 欄位1 排序方向, 欄位2 排序方向;
```

### 範例：先依城市排序，同城市再依姓名排序

```sql
SELECT name, city
FROM customers
ORDER BY city ASC, name ASC;
```

結果：
```
name    city
──────  ────
林雅婷  台中  ← 台中
周靜怡  桃園  ← 桃園
張宇航  台南  ← 台南
陳小明  台北  ← 台北，同城市再依姓名
劉思穎  台北
吳美玲  台北
黃建宏  新竹
王志偉  高雄
```

### 不同欄位使用不同排序方向

```sql
SELECT title, category_id, price
FROM products
ORDER BY category_id ASC, price DESC;
-- 先依分類升冪，同分類內依價格降冪
```

---

## 4.4 ORDER BY 使用欄位編號

可以用 `SELECT` 中的欄位位置編號代替欄位名稱：

```sql
SELECT title, price, stock
FROM products
ORDER BY 2 DESC;  -- 依第 2 個欄位（price）降冪
```

> ⚠️ 不建議這樣寫，可讀性差，修改 SELECT 欄位順序時容易出錯。

---

## 4.5 LIMIT — 限制回傳筆數

實際應用中常只需要前幾筆資料，用 `LIMIT` 限制：

```sql
SELECT 欄位 FROM 表格
ORDER BY 排序欄位
LIMIT 筆數;
```

### 範例：查詢最貴的 3 本書

```sql
SELECT title, price
FROM products
ORDER BY price DESC
LIMIT 3;
```

結果：
```
title     price
────────  ─────
黑暗森林  430
三體      420
海邊的卡夫卡 380
```

### 範例：庫存最少的 5 本書（庫存由少到多）

```sql
SELECT title, stock
FROM products
ORDER BY stock ASC
LIMIT 5;
```

---

## 4.6 OFFSET — 跳過前 N 筆（分頁）

`OFFSET` 搭配 `LIMIT` 可實現**分頁查詢**：

```sql
SELECT 欄位 FROM 表格
ORDER BY 排序欄位
LIMIT 每頁筆數 OFFSET 跳過筆數;
```

### 範例：書籍分頁，每頁顯示 3 本

```sql
-- 第 1 頁（第 1~3 筆）
SELECT title, price FROM products
ORDER BY product_id
LIMIT 3 OFFSET 0;

-- 第 2 頁（第 4~6 筆）
SELECT title, price FROM products
ORDER BY product_id
LIMIT 3 OFFSET 3;

-- 第 3 頁（第 7~9 筆）
SELECT title, price FROM products
ORDER BY product_id
LIMIT 3 OFFSET 6;
```

### 分頁公式

$$\text{OFFSET} = (\text{頁碼} - 1) \times \text{每頁筆數}$$

| 頁碼 | LIMIT | OFFSET |
|------|-------|--------|
| 1 | 3 | 0 |
| 2 | 3 | 3 |
| 3 | 3 | 6 |
| 第 N 頁 | 3 | (N-1) × 3 |

### 簡短寫法

SQLite 支援 `LIMIT 筆數, 偏移量` 的簡短格式（注意順序相反）：

```sql
-- 等同於 LIMIT 3 OFFSET 3
SELECT title FROM products LIMIT 3, 3;
```

> ⚠️ 建議使用 `LIMIT x OFFSET y` 的明確寫法，較為清晰。

---

## 4.7 ORDER BY + WHERE + LIMIT 組合

完整的查詢語句順序：

```sql
SELECT 欄位
FROM 表格
WHERE 條件
ORDER BY 排序欄位
LIMIT 筆數 OFFSET 偏移量;
```

### 範例：查詢庫存大於 0 的書，依價格由低到高，只取前 3 本

```sql
SELECT title, price, stock
FROM products
WHERE stock > 0
ORDER BY price ASC
LIMIT 3;
```

結果：
```
title     price  stock
────────  ─────  ─────
活著      280    60
傳說      290    15
挪威的森林 320   45
```

---

## 4.8 NULL 值在排序中的位置

SQLite 中，`NULL` 值在升冪（ASC）排序時出現在**最前面**：

```sql
SELECT title, published_year
FROM products
ORDER BY published_year ASC;
-- NULL 值的書排在最前面
```

若想讓 NULL 排在最後：

```sql
SELECT title, published_year
FROM products
ORDER BY published_year IS NULL ASC, published_year ASC;
-- published_year IS NULL 回傳 0 或 1，NULL 為 1（較大），排後面
```

---

## 重點整理

| 語法 | 用途 |
|------|------|
| `ORDER BY 欄位` | 升冪排序（預設） |
| `ORDER BY 欄位 ASC` | 升冪排序 |
| `ORDER BY 欄位 DESC` | 降冪排序 |
| `ORDER BY 欄位1, 欄位2 DESC` | 多欄位排序 |
| `LIMIT N` | 只回傳前 N 筆 |
| `LIMIT N OFFSET M` | 跳過 M 筆，取 N 筆（分頁） |

### SQL 子句完整順序

```sql
SELECT   -- 1. 選取欄位
FROM     -- 2. 來源表格
WHERE    -- 3. 篩選條件
ORDER BY -- 4. 排序
LIMIT    -- 5. 限制筆數
OFFSET   -- 6. 跳過筆數
```
