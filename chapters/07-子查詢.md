# 第 07 章：子查詢 {#ch07}

> **學習目標**
> - 理解子查詢的概念與使用時機
> - 在 WHERE 中使用子查詢
> - 在 FROM 中使用子查詢（衍生表格）
> - 在 SELECT 中使用子查詢
> - 使用 EXISTS 判斷是否存在

---

## 7.1 什麼是子查詢？

**子查詢**（Subquery）是指在一條 SQL 語句內部，嵌入另一條 `SELECT` 查詢。  
內層的查詢先執行，結果作為外層查詢的條件或資料來源。

```sql
SELECT ...
FROM ...
WHERE 欄位 = (SELECT ... FROM ...);
--            └──── 子查詢（先執行）────┘
```

---

## 7.2 WHERE 中的子查詢

### 單一值子查詢（= 運算子）

子查詢只回傳**一個值**時，用 `=` 比較：

```sql
-- 問：比「活著」這本書更貴的書有哪些？
SELECT title, price
FROM products
WHERE price > (
    SELECT price
    FROM products
    WHERE title = '活著'   -- 回傳 280
);
```

等同於 `WHERE price > 280`，但更靈活（書的價格可能改變）。

結果：
```
title             price
──────────────    ─────
挪威的森林        320
海邊的卡夫卡      380
單車失竊記        340
三體              420
哈利波特          360
恆毅力            350
傳說              290
黑暗森林          430
```

### 多值子查詢（IN 運算子）

子查詢回傳**多個值**時，用 `IN`：

```sql
-- 問：劉慈欣的所有書有哪些？
SELECT title, price
FROM products
WHERE author_id IN (
    SELECT author_id
    FROM authors
    WHERE name = '劉慈欣'   -- 回傳 author_id = 4
);
```

結果：
```
title     price
────────  ─────
三體      420
黑暗森林  430
```

## 7.3 NOT IN 排除

```sql
-- 問：哪些顧客從未下過訂單？
SELECT name
FROM customers
WHERE customer_id NOT IN (
    SELECT DISTINCT customer_id
    FROM orders
);
```

> ⚠️ **NOT IN 與 NULL 的陷阱**：若子查詢結果包含 NULL，`NOT IN` 會回傳空結果。  
> 遇到這種情況改用 `NOT EXISTS`（見 7.5 節）。

---

## 7.4 FROM 中的子查詢（衍生表格）

子查詢放在 `FROM` 中，作為一張「暫時的表格」（也叫**衍生表格**）：

```sql
SELECT 欄位
FROM (
    SELECT ...  -- 子查詢，產生暫時表格
    FROM ...
) AS 別名        -- 衍生表格必須設定別名
WHERE ...;
```

### 範例：查詢平均售價以上的書籍

```sql
SELECT title, price
FROM products
WHERE price > (
    SELECT AVG(price) FROM products
);
```

### 範例：每個分類中，最貴的書

```sql
SELECT p.title, p.price, p.category_id
FROM products p
JOIN (
    SELECT category_id, MAX(price) AS max_price
    FROM products
    GROUP BY category_id
) AS cat_max ON p.category_id = cat_max.category_id
           AND p.price = cat_max.max_price;
```

---

## 7.5 EXISTS — 判斷是否存在

`EXISTS` 檢查子查詢是否回傳**至少一列**，只要有，就為 `TRUE`：

```sql
SELECT 欄位
FROM 外層表格
WHERE EXISTS (
    SELECT 1   -- 慣例寫 1 或 *，只關心是否有結果
    FROM 內層表格
    WHERE 連接條件
);
```

### 範例：有下過訂單的顧客

```sql
SELECT name
FROM customers c
WHERE EXISTS (
    SELECT 1
    FROM orders o
    WHERE o.customer_id = c.customer_id
);
```

### NOT EXISTS — 不存在的情況

```sql
-- 從未下過訂單的顧客
SELECT name
FROM customers c
WHERE NOT EXISTS (
    SELECT 1
    FROM orders o
    WHERE o.customer_id = c.customer_id
);
```

## 7.6 SELECT 中的子查詢（純量子查詢）

在 `SELECT` 欄位位置放入子查詢，為每一列計算對應值：

```sql
SELECT
    title,
    price,
    (SELECT ROUND(AVG(price), 0) FROM products) AS 全站平均價,
    price - (SELECT AVG(price) FROM products)   AS 與均價差距
FROM products
ORDER BY price DESC;
```

結果：
```
title             price  全站平均價  與均價差距
──────────────    ─────  ──────────  ──────────
黑暗森林          430    335         95
三體              420    335         85
...
死水微瀾          260    335         -75
```

## 7.7 子查詢 vs JOIN

很多情況子查詢和 JOIN 可以互換，各有優缺點：

| 比較 | 子查詢 | JOIN |
|------|--------|------|
| 可讀性 | 邏輯直覺，好理解 | 需理解連接概念 |
| 效能 | 複雜子查詢可能較慢 | 通常較快 |
| 使用時機 | 篩選條件、聚合比較 | 需要顯示多表欄位 |

```sql
-- 子查詢寫法
SELECT title FROM products
WHERE author_id IN (
    SELECT author_id FROM authors
    WHERE nationality = '日本'
);

-- JOIN 寫法（同等效果）
SELECT p.title
FROM products p
JOIN authors a ON p.author_id = a.author_id
WHERE a.nationality = '日本';
```

---

## 重點整理

| 語法 | 用途 |
|------|------|
| `WHERE 欄位 = (子查詢)` | 與單一值比較 |
| `WHERE 欄位 IN (子查詢)` | 是否在子查詢結果中 |
| `WHERE 欄位 NOT IN (子查詢)` | 不在子查詢結果中（注意 NULL）|
| `FROM (子查詢) AS 別名` | 衍生表格 |
| `WHERE EXISTS (子查詢)` | 子查詢有結果則為 TRUE |
| `WHERE NOT EXISTS (子查詢)` | 子查詢無結果則為 TRUE |
| `SELECT (子查詢)` | 純量子查詢（每列計算）|
